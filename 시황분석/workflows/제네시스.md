# 🚀 제네시스(Genesis) v3.5 시황 분석 워크플로우

본 워크플로우는 네이버 증권 데이터를 직접 수집하여 **'3계층 입체 분석'**을 수행하고, 웹사이트 게시용 MDX 리포트를 자동 생성하는 전문 프로세스입니다.

---

## 1. 실행 트리거 (Trigger)

사용자가 **"제네시스 하자"** 또는 **"제네시스 시황 분석"** 입력 시 즉시 가동.

---

## 2. 분석 엔진 가동 (6-Step Engine)

### [Step 1] 실시간 데이터 스캔 (Data Sourcing)

> **데이터 수집은 반드시 `skills/market-data-miner-skill.md`의 8단계 체크리스트를 순서대로 실행한다.**
> WebFetch로 finance.naver.com 직접 접근 금지 (차단됨). 반드시 curl 사용.

**수집 항목 및 최우선 신뢰 소스:**

| 항목 | 소스 | 수집 방법 |
|------|------|----------|
| 코스피/코스닥 수급 (외국인·기관·개인 순매수) | 네이버 증권 투자자별 매매동향 **확정치** | `curl` → `m.stock.naver.com/api/index/{INDEX}/trend` |
| 코스피/코스닥 지수·등락률 | 네이버 증권 | `curl` → `m.stock.naver.com/api/index/{INDEX}/price` |
| 업종별 시세 (강세/약세 섹터) | 네이버 증권 업종별 시세 | `curl` + node EUC-KR 파싱 → `sise_group.naver?type=upjong` |
| 테마별 시세 (주도 테마) | 네이버 증권 테마 | `curl` + node EUC-KR 파싱 → `theme.naver` |
| 원/달러 환율·미 10년물 금리·WTI 유가 | Investing.com | `WebFetch` |
| 공포·탐욕 지수 (Fear & Greed) | CNN | `WebSearch` |

**데이터 검증 원칙:**
- 뉴스 기사·장중 속보의 수급 수치는 사용 금지. 네이버 확정치만 신뢰.
- 수집 후 `bizdate`가 오늘 날짜인지 반드시 확인.
- 모든 종목·테마 언급 시 **등락률(%) 반드시 병기**.

---

### [Step 2] 3계층 입체 분석 (3-Layer Analysis)

**[Layer 1] 거시 분석 (Macro)**
- 환율 변동성 및 거시 리스크(금리·유가·지정학)가 증시에 미치는 영향 분석
- 글로벌 자금 흐름과 달러 인덱스 방향성 진단

**[Layer 2] 중기 분석 (Mid-term)**
- 지수 지지·저항선 확인 및 바닥 여부 판단
- 외국인·기관 수급 쏠림 현상 및 개인 역할 반전 여부 진단
- 섹터 로테이션 패턴 — 자금이 어디서 어디로 이동하는지 추적

**[Layer 3] 단기 분석 (Short-term)**
- 당일 주도 섹터 및 **상한가/급등주(대장주)** 중심 수급 분석
- 등락률이 미미한 후발주자나 기대감뿐인 소외 테마는 철저히 배제
- 수급 + 등락률 두 조건이 동시에 확인된 종목만 언급

---

### [Step 3] 리스크 판정 (Risk Assessment)

- 현재 시장의 **결정적 상방 트리거** / **결정적 하방 트리거** 각각 명시
- 과매도·과매수 구간 판정 및 심리적 임계점 분석
- Fear & Greed 지수 기반 시장 온도 진단

---

### [Step 4] 오늘의 매매 전략 (Action Plan)

- **Do (해야 할 것):** 기관 추종, 분할 매수, 섹터 집중 등 구체적 지침.
  → **반드시 당일 수급·등락률이 확인된 구체적 대표 종목 명시**
- **Don't (하지 말아야 할 것):** 뇌동매매, 낙폭과대주 추격 금지 등.
  → **리스크가 감지되거나 하락 중인 구체적 종목을 사례로 경고**

---

### [Step 5] 안팀장의 한 줄 평 (Insight Synthesis)

전체 시황을 관통하는 촌철살인의 분석 멘트 한 줄 생성.
데이터 기반 + 날카로운 시각 + 전문 투자 전략가 톤.

---

### [Step 6] 보고서 최종 생성 및 배포 승인 (Push Control)

**저장 경로:** `content/market-analysis/YYYY-MM-DD-market-analysis-genesis.mdx`

**MDX 프론트매터 필수 항목 (누락 시 웹사이트 미노출):**
```yaml
title: "..."          # 핵심 이슈 중심의 임팩트 있는 제목
date: "YYYY-MM-DD"
category: "Genesis"
summary: "..."        # 2~3줄 요약
tags: [...]           # 관련 키워드 배열
thumbnail: "..."      # Unsplash URL
```

**GitHub Push 규칙:** 황원장님의 명확한 승인("Go", "Push해줘") 이후에만 실행.
Push 절차는 `workflows/push.md` 참고.

---

## 3. 리포트 스타일 가이드

- **어조:** 신뢰감 있고 날카로운 전문 투자 전략가 톤. 과장 없이 데이터로 말한다.
- **데이터 원칙:** 모든 수치에 출처 명시. 추정치와 확정치 구분 표기.
- **시각화:** 표, 인용구(`>`), 알림창(`[!IMPORTANT]`) 적극 활용.
- **섹터 언급:** 등락률(%) 병기 필수. 수급 확인된 대장주 함께 명시.

---

## 4. 실행 스크립트 연결

```
Step 1 데이터 수집  →  skills/market-data-miner-skill.md (8단계 전 자동화)
Step 2~5 분석      →  3-Layer 프레임워크 + Risk + Action + 한 줄 평
Step 6 저장        →  content/market-analysis/YYYY-MM-DD-market-analysis-genesis.mdx
Step 6 Push        →  workflows/push.md (황원장님 승인 후)
```
