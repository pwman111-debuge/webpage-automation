# 🚀 한국 주식 장기 유망 종목 발굴 워크플로우 (Genesis)

이 워크플로우는 한국 시장에서 1년~3년의 호흡으로 안정적인 성장을 기록할 '장기 유망 종목'을 발굴하기 위한 안티그래비티 AI의 공식 방법론입니다.

## 📋 핵심 투자 철학
- **기간 설정:** 12개월 ~ 36개월 (산업의 실적이 숫자로 증명되는 기간)
- **선별 기준:** 
  - **ROE(자기자본이익률) 15% 이상**의 꾸준한 수익성
  - 글로벌 시장 점유율 및 **독보적 해자(Moat)** 보유 여부
  - 인구 구조, 정책, 기술 변화에 따른 **구조적 성장주**

## 🛠 실행 단계

// turbo
1. **시장 환경 및 매크로 분석 (Top-Down 진입)** `🔍 Deep-Dive Mode`

   "제네시스" 명령어가 입력되면 개별 종목 스크리닝에 앞서 아래 매크로 환경을 먼저 파악합니다. 개별 종목 아이디어가 아무리 좋아도 시장 방향성과 섹터 흐름이 역행하면 장기 수익률이 훼손되기 때문입니다.

   > **[Deep-Dive Mode 발동]** 이 단계는 요약본 의존을 금지합니다. 원본 소스 3개 이상 직접 확인 + 교차 검증 로그를 남긴 후 다음 단계로 자동 진행합니다. (`rules/rule.md` 참조)

   **1-1. 지수 및 밸류에이션 환경**
   - 네이버 증권에서 코스피·코스닥 현재 지수 위치 확인
   - 코스피 12개월 선행 PER이 역사적 평균(약 11~12배) 대비 고평가·저평가 여부 판단
   - 공포·탐욕 구간 파악: 52주 신고가 종목 비율, VIX 한국판(VKOSPI) 수준 참조

   **1-2. 금리·환율·정책 변수**
   - 한국은행 기준금리 방향성 (인상·동결·인하 사이클 어느 구간인지)
   - 원/달러 환율 추세 — 강달러 환경 시 수출주 유리, 약달러 시 내수·바이오 유리
   - 현 정부의 핵심 정책 테마 (예: 반도체 특별법, 에너지 전환, 저출생 대응 등) 파악

   **1-3. 구조적 성장 섹터 사전 선정**
   - 위 매크로 분석 결과를 바탕으로 향후 1~3년 수혜가 기대되는 **섹터 2~3개**를 먼저 확정
   - 섹터 선정 기준: ① 정책 수혜 ② 인구·기술 구조 변화 ③ 글로벌 공급망 재편
   - 이후 2단계 스크리닝은 선정된 섹터 내 종목으로 범위를 한정하여 신호 대 잡음비를 높임

2. **퀀트 스크리닝 — 후보군 압축 (Funnel 방식)**

   1단계에서 확정된 섹터 내 종목을 대상으로 **`Genesis-Quant-Skill`**을 실행하여 3단계 퍼널로 후보군을 압축합니다. 데이터 소스는 **네이버 증권(메인)**이며, 수치 검증 시 토스 증권·구글 파이낸스를 보조로 활용합니다.

   **스킬 실행 명령어:**
   ```bash
   # 섹터 지정 스캔 (1단계에서 확정된 섹터명 입력)
   python -X utf8 skills/genesis-quant-skill/scripts/gene-scan.py --sector {섹터명} --top 20

   # 특정 종목 직접 분석
   python -X utf8 skills/genesis-quant-skill/scripts/gene-scan.py --tickers {코드1} {코드2}
   ```

   **네이버 증권 접속 방식 및 장애 대응:**
   - `gene-scan.py`는 브라우저 없이 **로컬 PC에서 직접 HTTP 요청**으로 네이버 증권 데이터를 수집합니다.
   - 네이버 증권 직접 접속이 차단되거나 응답이 없을 경우 → **FinanceDataReader(FDR)** 보조 소스로 자동 전환됩니다.
   - 그 외 네트워크 오류·타임아웃 발생 시 해당 종목은 "데이터 없음"으로 건너뛰고 스캔을 계속 진행합니다.
   - 재시도가 필요한 종목은 `--tickers {코드}` 옵션으로 개별 재실행합니다.

   **2-1. 1차 필터 — 재무 건전성 (Pass/Fail)**
   - 최근 3개년 매출·영업이익 **연속 우상향** 여부
   - ROE **15% 이상** (최근 연간 기준, 네이버 증권 > 종목 > 재무분석 탭)
   - 부채비율 **200% 이하** (금융주 제외)
   - 영업이익률 **8% 이상** — 원가 경쟁력 확인
   - 위 4개 조건 중 3개 이상 통과한 종목만 다음 단계로 진입

   **2-2. 2차 필터 — 성장성 및 밸류에이션**
   - 최근 4분기 누적 EPS 성장률 **YoY 10% 이상**
   - PER: 동종 섹터 평균 대비 **0.5배~1.2배** 구간 (저평가~적정가)
   - PBR: **2.0 이하** (간이 기준 — `gene-scan.py` 자동 적용) / 역사적 밴드 데이터 있을 경우 하단 50% 이내로 강화
   - 시가총액 **500억 이상** — 유동성 리스크 제거

   **2-3. 3차 필터 — 해자(Moat) 정성 검증 (에이전트 수동 분석)** `🔍 Deep-Dive Mode`
   > `gene-scan.py` 자동화 범위 밖 — 에이전트가 직접 판단합니다.
   > **[Deep-Dive Mode 발동]** IR 자료·공시 원문·뉴스 기사 등 원본 소스 3개 이상 직접 확인 필수.
   - 해당 종목이 속한 산업 내 **점유율 1~2위** 또는 독점적 기술·특허 보유 여부
   - 경쟁사 진입 장벽: 설비·인증·브랜드·네트워크 효과 중 하나 이상 존재
   - 최근 IR 자료·공시(네이버 증권 > 공시 탭)에서 중장기 성장 가이던스 확인
   - 3차 필터 통과 종목을 **최종 후보 5~8개**로 압축하여 3단계로 이관

3. **심층 수급 및 심리 분석 — 매집 구조 확인** `🔍 Deep-Dive Mode`

   > **[Deep-Dive Mode 발동]** 수급·심리 데이터는 요약 의존 시 오판 위험이 가장 높은 단계입니다. 네이버 증권 투자자별 매매동향·공시 원문 등 원본 소스 3개 이상 직접 확인 + 교차 검증 로그 필수.

   2단계에서 압축된 후보 5~8개를 대상으로 **"돈이 실제로 들어오고 있는가"**를 검증합니다. 아무리 좋은 기업도 수급이 이탈 중이면 장기 보유 시 기회비용이 발생합니다.

   **시계열 데이터 자동 수집 (스킬 연동):**
   ```bash
   # 52주 위치·PBR 밴드·차트·JSON 자동 생성
   python -X utf8 skills/genesis-quant-skill/scripts/gene-scan.py --tickers {후보코드들} --detail
   ```
   - 출력되는 `genesis_detail.json`에서 52주 위치, PBR 밴드 하단 50% 저평가 여부를 바로 확인합니다.
   - 차트 PNG는 시각적 검증에 활용합니다.

   **⚠️ 3단계 결과 보고서 반영 의무 (5단계 연결):**
   - `genesis_detail.json`의 핵심 값은 **반드시 5단계 보고서의 "수급 현황" 파트에 그대로 기재**해야 합니다.
   - 스크립트를 돌리기만 하고 결과를 보고서에 반영하지 않는 것은 **3단계 미완료**로 간주합니다.
   - 필수 기재 항목:
     1. **52주 위치(%)** — 30% 이하일 경우 "안전마진 구간", 70% 이상일 경우 "고점 부근" 명시
     2. **PBR 밴드 저평가 여부** — 현재 PBR vs 3년 중앙값 대비 저평가/적정/고평가 판정
     3. **현재 PBR / 3년 평균 PBR / 3년 최저 PBR** 수치 병기

   **3-1. 외국인·기관 수급 추이 (네이버 증권 > 투자자별 매매동향)**
   - 최근 3개월·6개월 외국인 순매수/순매도 방향성 확인
   - 기관(투신·연기금·보험) 중 **연기금의 순매수 지속 여부** — 장기 스마트머니 신호
   - 외국인 보유 비중이 **역사적 저점** 부근이면 추가 매집 여력으로 해석
   - 개인 과매수 구간(단기 급등 + 외국인·기관 동반 매도)은 진입 보류

   **3-2. 주가 위치 및 기술적 구조**
   - 현재 주가가 **52주 저점 대비 30% 이내** 구간인지 확인 — 장기 매수 안전마진
   - 120일·200일 이동평균선 배열 상태: 정배열 전환 초기 구간이 이상적
   - 거래량 패턴: 주가 하락 시 거래량 감소(매물 소화), 반등 시 거래량 증가 구조 선호
   - **역사적 PBR 밴드 하단 50% 이내** 재확인 (2단계 수치와 교차 검증)

   **3-3. 투자심리 및 시장 내러티브**
   - 네이버 증권 뉴스·공시 탭에서 최근 3개월 내 **부정적 이벤트**(소송, 횡령, 대규모 유상증자) 유무 점검
   - 해당 섹터가 현재 시장의 **관심 밖(소외)**인지 — 소외 섹터일수록 역발상 기회
   - 애널리스트 목표주가 컨센서스 대비 현재 주가 괴리율 확인: **괴리율 30% 이상**이면 리레이팅 여력으로 평가
   - 3단계 분석 결과를 종합해 후보군을 **최종 3~5개**로 압축하여 4단계로 이관

4. **최종 종목 선정 및 투자 설계 (필수 포함 사항)** `🔍 Deep-Dive Mode`

   > **[Deep-Dive Mode 발동]** 목표가·손절가 산출 근거는 반드시 원본 재무 데이터 기반이어야 합니다. 추정치·요약본 사용 금지. 산출 근거 로그를 남기고 5단계로 자동 진행합니다.

   3단계까지 살아남은 후보 중 **최종 3종목**을 확정하고, 종목별 투자 설계를 완성합니다. 숫자 없는 선정은 의견이지 분석이 아닙니다.

   **4-1. 최종 3종목 확정 및 선정 근거 요약**
   - 각 종목에 대해 **"왜 지금인가"** 한 문장으로 핵심 투자 thesis 정리
   - 1단계 섹터 논리 → 2단계 퀀트 → 3단계 수급까지 일관된 스토리 연결 여부 검토
   - 3종목 간 섹터·스타일 중복 최소화 (동일 섹터 2종목 이상 편입 지양)

   **4-2. 목표가 산출 (2가지 방법 병행)**
   - **PER 밸류에이션:** 향후 2년 예상 EPS × 적정 PER (섹터 평균 또는 역사적 고점 PER) = 목표가
   - **PBR 밴드 상단:** 역사적 PBR 상단 배수 × 현재 BPS = 상방 시나리오 목표가
   - 두 방법의 결과를 평균하거나 보수적 값을 채택하여 **12개월·24개월·36개월 목표가** 각각 제시
   - 현재가 대비 기대 수익률 명시 (예: +45% / 24개월)

   **4-3. 손절가 및 투자 아이디어 훼손 조건**
   - **기술적 손절:** 매수 시점 기준 **-15% 이탈** 시 포지션 재검토
   - **펀더멘털 손절 트리거** (아래 중 하나라도 발생 시 즉시 재검토):
     - 연간 영업이익이 전년 대비 **20% 이상 역성장**
     - 핵심 경영진 교체 또는 지배구조 리스크 발생
     - 투자 thesis의 전제가 된 정책·기술 환경이 180도 변화
   - 손절가는 "얼마에 팔 것인가"가 아닌 **"어떤 조건이 깨지면 아이디어가 틀린 것인가"** 기준으로 설정

   **4-4. 포지션 사이징 가이드**
   - 3종목 합산 비중: 포트폴리오의 **최대 30~40%** 이내 권고 (장기 투자 특성상 집중도 조절)
   - 종목별 비중: 확신도 순으로 **메인 15% / 서브 10% / 관찰 5%** 구조 제안
   - 분할 매수 원칙: 목표 비중의 **50% 즉시 진입 + 50% 조정 시 추가 매수**

5. **보고서 생성 및 표준화 (Genesis SOP)**

   4단계까지의 분석을 **재현 가능한 프리미엄 리포트**로 압축합니다. 읽는 사람이 분석 과정을 다시 따라갈 수 있어야 하며, 숫자·근거·판단이 명확히 분리되어야 합니다.

   **5-1. 리포트 구조 (고정 템플릿)**
   ```
   ## 📌 매크로 요약 (1단계 요약)
   - 시장 환경 한 줄 진단 + 선정 섹터 및 근거

   ## 🔍 종목별 분석 (2~3단계 요약, 종목당 반복)
   ### [종목명] (티커)
   - 핵심 투자 Thesis (1~2문장)
   - 재무 지표 요약표 (Markdown 표 — `--report-data` 출력 그대로)
   - **시계열 분석 (3단계 `--detail` JSON 필수 반영):**
     - 52주 위치: {N}% ({안전마진/중립/고점부근} 판정)
     - PBR 밴드: 현재 {X}x / 3년 평균 {Y}x / 3년 최저 {Z}x → {저평가/적정/고평가}
   - 수급 현황 요약 (외국인·연기금 등)
   - 리스크 요인

   ## 🎯 투자 설계 요약표 (4단계 요약)
   | 종목 | 현재가 | 목표가(12M) | 목표가(24M) | 손절가 | 기대수익률 |
   |------|--------|------------|------------|--------|-----------|

   ## 📊 포지션 사이징 (4-4 반영 의무)
   - 메인 {종목}: 15% / 서브 {종목}: 10% / 관찰 {종목}: 5%
   - 분할 매수: 목표 비중의 50% 즉시 + 50% 조정 시 추가

   ## ⚠️ 공통 리스크
   - 시장 전반 리스크 및 포트폴리오 차원 유의사항
   ```

   **5-2. 재무 지표 표준 양식 (종목별 필수 포함)**

   > 연도는 **보고서 작성 시점 기준으로 동적 산출**합니다.
   > 규칙: `최근 확정 연도 기준 -2년 | -1년 | 최근 확정 | 차년도(E)`
   > 예시 (2026년 4월 기준, 2025년 결산 완료된 경우): **2023 | 2024 | 2025 | 2026E**
   > 예시 (2026년 4월 기준, 2025년 결산 미완료): **2022 | 2023 | 2024 | 2025E**
   > **네이버 증권 재무분석 탭에서 가장 최근 확정 연도를 직접 확인 후 적용합니다.**

   | 항목 | {N-2}년 | {N-1}년 | {N}년 | {N+1}년(E) |
   |------|---------|---------|-------|-----------|
   | 매출액 (억원) | | | | |
   | 영업이익 (억원) | | | | |
   | 영업이익률 (%) | | | | |
   | ROE (%) | | | | |
   | PER (배) | | | | |
   | PBR (배) | | | | |

   **5-3. 작성 원칙**
   - 문장은 **간결하고 단정하게** — 수식어 남발 금지, 팩트 중심
   - 모든 수치는 **출처(네이버 증권 기준일)** 병기
   - **데이터 출처는 네이버 증권 단독** — DART·증권사 컨센서스·기타 포털 수치 혼용 금지
   - **현재가는 보고서 작성 시점에 네이버 증권 실시간 시세를 직접 확인**하여 기재 (추정·기억값 사용 금지)
   - 재무 수치는 반드시 `gene-scan.py` 출력값 또는 네이버 증권 재무분석 탭 원본과 대조 후 기재
   - Markdown 표 앞뒤 **빈 줄 1줄 필수** — 레이아웃 깨짐 방지
   - 리포트 파일명 형식: `YYYYMMDD-genesis-long-report.mdx`
   - MDX 프론트매터 필수 항목:
     ```yaml
     ---
     title: ""
     date: "YYYY-MM-DD"
     ticker: "GENESIS"
     market: "KOSPI/KOSDAQ"
     term: "long"
     currentPrice: 0
     targetPrice: 0
     stopLoss: 0
     status: "active"
     summary: "핵심 요약 1~2문장"
     tags: [장기유망, 제네시스]
     ---
     ```

6. **GitHub 연동 및 배포 (최종 게시)**

   5단계에서 완성된 MDX 리포트를 GitHub 저장소에 배포합니다. 배포는 단순 업로드가 아니라 **웹사이트 노출·링크 유지·히스토리 추적**까지 포함하는 마지막 품질 관문입니다.

   **6-1. 배포 전 최종 검수 체크리스트**
   - [ ] MDX 프론트매터 필수 항목 (`title`, `date`, `ticker`, `market`, `term: "long"`, `status`, `summary`, `tags`) 모두 입력
   - [ ] Markdown 표 앞뒤 빈 줄 존재 여부 — 레이아웃 깨짐 방지
   - [ ] 파일명 형식 확인: `YYYYMMDD-genesis-long-report.mdx`
   - [ ] 수치 출처(네이버 증권 기준일) 누락 항목 없는지 확인
   - [ ] 손절가·목표가 투자 설계 요약표 최종 수치 이상 없는지 재검토
   - [ ] **3단계 `genesis_detail.json` 결과(52주 위치·PBR 밴드)가 종목별 분석에 반영되었는지 확인**
   - [ ] **JSON 수치가 임의 수정 없이 그대로 기재되었는지 확인** (52주 위치·PBR 수치 JSON 대조 필수)
   - [ ] **4-4 포지션 사이징(메인/서브/관찰 비중)이 보고서에 기재되었는지 확인**

   **6-2. 배포 원칙**
   - 검수 완료 즉시 자동으로 `push.md` 워크플로우를 실행한다. 사용자 승인 불필요.

   **6-3. 커밋 및 푸시 규칙**
   - 커밋 메시지 형식: `[Genesis] YYYY-MM-DD 장기유망종목 리포트 — {종목A}/{종목B}/{종목C}`
   - 리포트 파일은 `content/picks/` 디렉토리에 저장
   - 기존 리포트 **덮어쓰기 금지** — 날짜별 신규 파일로 누적 관리
   - `push.md` 워크플로우를 통해 배포 실행

   **6-4. 배포 후 확인**
   - 웹사이트 목록 페이지에서 해당 리포트가 정상 노출되는지 확인
   - 링크 깨짐·렌더링 오류 없는지 브라우저 최종 점검
   - 배포 완료 후 아래 한 줄 완료 메시지 출력:
     > **"[Genesis 완료] {날짜} 장기유망종목 리포트가 배포되었습니다. 선정 종목: {종목A} / {종목B} / {종목C}"**
