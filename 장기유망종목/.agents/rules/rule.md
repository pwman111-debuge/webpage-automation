## 🔍 Deep-Dive Mode (강제 심층 검증 모드)

### 발동 조건
워크플로우 실행 중 아래 단계에 진입하면 **자동으로 Deep-Dive Mode가 발동**됩니다:
- **1단계**: 매크로 환경 분석 (지수·금리·정책 판단)
- **2단계 2-3**: 해자(Moat) 정성 검증
- **3단계**: 수급·심리 분석 전체
- **4단계**: 최종 종목 선정 및 목표가 산출

### 핵심 원칙 — 반드시 지켜야 할 3가지

1. **원본 소스 3개 이상 직접 확인 의무**
   - 검색 결과 요약본·스니펫에 의존하여 판단을 내리는 것을 금지합니다.
   - 반드시 **원본 페이지(네이버 증권 종목 페이지, 공시 원문, IR 자료 등)를 직접 열어서 수치를 확인**해야 합니다.
   - 각 판단 근거마다 **출처 URL 또는 탭 경로를 명시**합니다. (예: `네이버 증권 > 005930 > 재무분석 탭 > 2024.12 기준`)

2. **교차 검증 의무**
   - 하나의 수치를 채택하기 전, **최소 2개 이상의 소스에서 동일 수치를 확인**해야 합니다.
   - 소스 간 수치가 다를 경우 차이 이유를 분석하고, 더 신뢰할 수 있는 근거를 명시합니다.
   - 단순히 "일치한다"가 아닌 **"A 소스에서 X, B 소스에서 X로 일치 확인"** 형식으로 기록합니다.

3. **조기 종료 금지**
   - 분석이 충분하지 않은 상태에서 "결론을 내기에 충분한 정보를 확보했다"고 자의적으로 판단하여 단계를 건너뛰는 것을 금지합니다.
   - 원본 소스 3개 이상 직접 확인 + 교차 검증 요건이 충족되지 않으면 다음 단계로 넘어가지 않습니다.

### Deep-Dive 단계 진행 방식
- **각 단계 완료 시 승인을 요청하지 않고 자동으로 다음 단계로 진행합니다.**
- 단, 각 단계에 들어갈 때 아래 형식으로 확인 사항을 간단히 로그로 남깁니다:

```
[Deep-Dive 로그 — {단계명}]
- 확인한 원본 소스:
  1. {소스명} — {URL 또는 경로} — {확인 항목}
  2. {소스명} — {URL 또는 경로} — {확인 항목}
  3. {소스명} — {URL 또는 경로} — {확인 항목}
- 교차 검증 결과: {일치 여부 및 차이 내용}
```

- 최종 배포(6단계) 시에만 황원장의 명시적 승인을 받습니다.

### Deep-Dive Mode에서 허용되지 않는 행동
- 검색 결과 첫 페이지 요약만 읽고 수치 인용
- "일반적으로 알려진 바에 따르면..." 형식의 선언
- 출처 없는 수치 제시
- 원본 소스 3개 미만으로 확인하고 단계 완료 처리
- Deep-Dive 로그 없이 단계 진행

## 📊 데이터 출처 단일화 원칙 (네이버 증권 단독)

### 핵심 규칙
- **모든 재무 수치·주가·밸류에이션 지표는 네이버 증권 데이터만 사용**합니다.
- DART 공시, 증권사 컨센서스, 구글 파이낸스, 토스 증권 등 **타 소스 수치를 보고서에 혼용 금지**.
- 보조 소스는 **검증 목적으로만** 참조 가능하며, 보고서 본문 수치로는 채택하지 않습니다.

### 보고서 작성 시 필수 절차
보고서에 기재할 종목별 재무 데이터는 반드시 아래 명령어를 먼저 실행하여 네이버 증권 원본을 수집합니다:

```bash
python -X utf8 skills/genesis-quant-skill/scripts/gene-scan.py \
    --tickers {종목1} {종목2} {종목3} --report-data
```

이 명령의 출력물을 **그대로 보고서에 붙여넣습니다**. 추정·기억·요약으로 수치를 작성하지 않습니다.

### 연도 표기 원칙
- 재무표 연도는 **보고서 작성 시점의 네이버 증권 재무분석 탭에 표시된 연도를 그대로 사용**합니다.
- 하드코딩된 연도(예: 2022~2025E) 사용 금지 — `gene-scan.py --report-data` 출력의 연도 헤더를 그대로 따릅니다.

### 현재가 기재 원칙
- **반드시 보고서 작성 시점에 `--report-data`로 실시간 조회한 가격을 사용**합니다.
- 과거 기억·추정치·다른 날짜의 가격 사용 금지.

### 위반 시 대응
- 황원장이 보고서에서 네이버 증권과 다른 수치를 발견할 경우, 해당 보고서는 **전면 재작성**합니다.
- 재발 시 해당 종목 분석은 처음부터 `--report-data` 출력만 사용하여 작성합니다.

## 🔒 스크립트 산출물 원본 보존 원칙

### 핵심 규칙
- `gene-scan.py --detail`로 생성된 **`genesis_detail.json`의 수치는 임의 수정 없이 그대로 보고서에 기재**합니다.
- **수치 해석(저평가/고점 등 판정 문구)은 자유롭게 작성 가능**하지만, **숫자 자체를 변경·축소·확대하는 것은 금지**합니다.

### 대상 항목 (절대 변경 금지)
- `52w.52주위치(%)` — 소수점 한 자리까지 그대로
- `52w.현재가`, `52w.52주고점`, `52w.52주저점`
- `pbr_band.현재PBR`, `pbr_band.3년평균PBR`, `pbr_band.3년최저PBR`, `pbr_band.3년최고PBR`, `pbr_band.밴드하단50%`

### 해석(판정 문구) 작성 원칙
아래와 같이 **수치는 JSON 그대로, 판정 문구만 맥락에 맞게** 작성합니다:

```
52주 위치: 93.4% (JSON 원본)
  → 판정: "고점 부근, 단기 조정 리스크 존재"   ← 해석은 자유
  → 금지: "72.5% (중립 구간)"                  ← 수치 변경은 위반
```

### 위반 유형 및 대응
- **스토리에 맞춰 수치 임의 조정** — 발견 시 해당 종목 분석 전면 재작성.
- **JSON 값과 보고서 값이 다른 경우** — 배포 불가, 황원장 재검토 요청.
- 판정 문구가 실제 수치와 모순되는 경우 (예: 52주 93%인데 "저평가 구간") — 해석 오류로 재작성.
