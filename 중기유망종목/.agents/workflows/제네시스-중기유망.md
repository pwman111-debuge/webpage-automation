# 제네시스(Genesis) 워크플로우: 중기 유망 종목 발굴

이 워크플로우는 **네이버 증권을 메인 소스**로 활용하여 **중기(3~6개월)** 단위의 유망 종목을 발굴하고, **목표가·손절가·수급 분석·성과 추적**이 포함된 심층 분석 보고서를 생성합니다.

> **데이터 소스 원칙**
> - **메인:** 네이버 증권 (시장지표, 업종, 테마, 종목분석, 투자의견, 차트, 공시, 리서치, 투자자별 매매동향, 공매도 현황)
> - **보조:** 구글 파이낸스 / 토스 증권 (글로벌 변수 확인, 커뮤니티 보조 참고 목적만)
>
> **⚙️ 데이터 수집 방법**
> 네이버 증권은 WebFetch 직접 접근이 차단되므로, 로컬 Python 스크립트로 실시간 데이터를 수집한다.
> - 스크립트: `scripts/naver_finance.py`
> - 스킬 가이드: `.agents/skills/genesis-data.md` ← **워크플로우 시작 전 반드시 숙지**
>   (명령별 출력 해석 방법, 주의사항, 브라우저 직접 확인 링크 포함)

## 실행 트리거
사용자가 **"제네시스 하자"** 또는 **"/genesis"** 라고 요청하면 이 워크플로우를 시작합니다.

---

## 단계별 프로세스

### 1단계: 시장 및 섹터 스캔 (Macro Scan)

> **목표:** 현재 시장 국면을 정확히 진단하고, 향후 3~6개월 모멘텀이 살아있는 섹터 2~3개를 선정한다.

#### 1-1. 국내 시장 전체 체온 측정
```bash
python scripts/naver_finance.py market
```
- **KOSPI / KOSDAQ 지수 흐름:** 20일·60일 이평선 위/아래 여부, 최근 5거래일 방향성 확인
- **외국인·기관 전체 시장 순매수 동향:** 네이버 증권 [투자자별 시장 매매동향]에서 최근 2주 기준 자금 유입 방향 파악
- **시장 국면 판단 (4가지 중 하나로 정의):**
  - `상승추세 초입` / `상승추세 지속` / `조정 중` / `하락추세`
  - → 판단 결과를 이후 섹터·종목 선정 기준의 **리스크 조절 근거**로 활용

#### 1-2. 주도 섹터 선별
```bash
python scripts/naver_finance.py sector
```
- 출력: 업종명 / 등락률 / 상승↑·하락↓·전체 종목 수 / 업종번호
- 업종별 시세에서 **등락률 상위 업종** 5개 추출 후 아래 기준으로 **지속 가능한 섹터 2~3개**로 압축:
  - 단순 단기 급등이 아닌, **거래대금 증가 + 외국인/기관 동반 유입** 여부로 진성 모멘텀 구분
  - 정책·수출·기술 트리거가 있는가? (단순 테마성 제외)
  - 실적 시즌(분기 실적 발표)과 연동되는가?
  - 글로벌 동종 업종과 같은 방향인가?
- **업종번호는 2단계 `screen` 명령에 그대로 사용**

#### 1-3. 테마 모멘텀 교차 확인
```bash
python scripts/naver_finance.py theme
```
- 출력: 테마명 / 등락률 순위
- 1-2에서 선별한 섹터와 **테마가 겹치는 종목군**이 있는지 교차 검증 → 겹칠수록 모멘텀 신뢰도 높음
- 현재 시장에서 부각되는 **정책 테마**(정부 예산·규제 완화 등) 및 **기술 테마**(AI, 로봇, 바이오 등) 확인

#### 1-4. 증권사 리포트 트렌드 확인 (네이버 증권 [리서치] 탭 직접 참조)
- 브라우저: `https://finance.naver.com/research/company_list.naver`
- 최근 1~2주 내 **목표주가 상향 + BUY 의견 신규 발행** 리포트가 집중되는 섹터 파악
- 같은 섹터에 복수 증권사가 동시에 커버리지를 늘리는 경우 → **기관 선점 진입 신호**로 해석

#### 1-5. 글로벌 변수 보조 체크
- **달러/원 환율:** 원화 강세·약세가 수출주·내수주 중 어느 쪽에 유리한지 판단
- **미 증시 주요 섹터 (보조):** 반도체·AI·바이오 등 국내 동종 섹터에 영향을 주는 글로벌 흐름 확인
- 글로벌 변수는 **국내 섹터 선택을 보강하는 근거**로만 활용하며, 단독 선정 기준으로 삼지 않음

#### 1-6. 1단계 출력 (다음 단계 입력값)
1단계 완료 후 반드시 아래 항목을 정리하여 2단계로 넘긴다:
- **현재 시장 국면:** (예: "조정 중 / 저점 탐색 구간")
- **선정 섹터 2~3개 + 업종번호:** (예: 방산(no.274), 바이오(no.299), 2차전지소재(no.278))
- **선정 근거 한 줄 요약:** 섹터별로 지속 모멘텀의 핵심 트리거 명시
- **리스크 경고 (있을 경우):** 단기 과열 징후, 매크로 불확실성 등

---

### 2단계: 종목 스크리닝 및 펀더멘털 분석 (Deep Dive)

> **목표:** 1단계에서 선정된 섹터 내 종목을 정량·정성 기준으로 압축하여, 3단계(수급 분석)에 넘길 **후보 종목 5~7개**를 선별한다.

#### 2-1. 1차 정량 필터 — 업종 내 종목 풀 확보
```bash
python scripts/naver_finance.py screen [1단계에서 받은 업종번호]
# 예: python scripts/naver_finance.py screen 267
```
- 출력된 종목 리스트에서 아래 기준으로 **기계적으로 걸러낸다:**
  - 시가총액 **500억 이상** (유동성 확보)
  - 증권사 투자의견 커버리지 **2개사 이상** (기관 관심 최소 기준)
  - 증권사 평균 목표주가 대비 현재가 **괴리율 +15% 이상** (상승 여력 존재)
  - 최근 분기 영업이익 **흑자** (적자 기업 제외)
- 위 기준을 모두 통과한 종목만 2-2 이후 분석 대상으로 진입

#### 2-2. 실시간 시세 확보 (Critical — 반드시 이행)
```bash
python scripts/naver_finance.py stock [종목코드]
# 예: python scripts/naver_finance.py stock 005930
```
- 출력의 **★ 현재가 기준 시각 확정** 줄을 보고서에 그대로 기재
- 리포트 내 과거 시세, 전일 종가와 절대 혼동하지 않도록 주의
- **권리변경 확인:** 최근 1주일 이내 아래 공시 여부 반드시 체크
  - 액면분할 / 유·무상증자 / 거래정지 / 관리종목 지정
  - 해당 공시가 있는 종목은 분석 목록에서 즉시 제외
  - 공시 확인: `https://dart.fss.or.kr/dsab007/main.do?option=S&textCrpNm={회사명}`

#### 2-3. 펀더멘털 심층 분석
```bash
# 종목 기본 + 재무정보 + 목표주가 한 번에
python scripts/naver_finance.py stock [종목코드]

# 브라우저 직접 확인 (더 상세한 재무 데이터)
# https://finance.naver.com/item/main.naver?code={종목코드}
# https://finance.naver.com/item/coinfo.naver?code={종목코드}&target=cnc
```
각 항목을 **수치 기준과 함께** 확인한다:

**① 실적 성장성**
- 최근 4분기 누적 영업이익 YoY 성장률 → **+10% 이상** 우선
- 다음 분기 컨센서스 영업이익 방향 → 상향 추세인지 확인
- 최근 실적 발표에서 컨센서스 대비 **어닝 서프라이즈(+5% 초과)** 여부

**② 재무 건전성**
- ROE **10% 이상** (자본 효율성 기본 충족)
- 부채비율 **150% 이하** (과도한 레버리지 제외, 업종 특성 고려)
- 유동비율 **100% 이상** (단기 유동성 안전 여부)

**③ 밸류에이션**
- PER: 섹터 평균 대비 **할인 상태** 혹은 성장률 대비 저평가(PEG < 1) 여부
- PBR: 역사적 밴드 하단에 위치하는지 확인
- 증권사 평균 목표주가와 현재가의 **괴리율 재확인** (목표주가 신뢰도: 최근 3개월 내 발행 리포트 기준)

#### 2-4. 기술적 위치 확인 (네이버 증권 [차트] 탭)
- **이동평균선:** 현재가가 20일선·60일선 **위에 있는지** 확인 (추세 방향 일치 여부)
- **지지·저항:** 현재가가 주요 지지 구간(전 고점, 120일선 등) 근처인지 → 손절가 설정 근거
- **제외 조건:** 단기 급등으로 RSI 70 초과 상태 or 전고점 대비 -40% 이상 하락 중 종목은 이유 명시 후 보류 처리

#### 2-5. 촉매(Catalyst) 발굴
```bash
# 공시·뉴스 직접 확인 링크
# DART 공시:  https://dart.fss.or.kr/dsab007/main.do?option=S&textCrpNm={회사명}
# 종목 뉴스:  https://finance.naver.com/item/news_news.naver?code={종목코드}
# 종목 공시:  https://finance.naver.com/item/news_notice.naver?code={종목코드}
```
향후 3~6개월 내 주가를 움직일 **구체적 이벤트**가 있는지 확인:
- 대규모 수주 계약 / 신규 사업 진출 공시
- 설비 증설·공장 착공 일정
- 예정된 실적 발표 및 기대치 상회 가능성
- 정부 정책 수혜 (예산 배정, 인허가 등)
- **촉매가 없는 종목**은 후보군 하위로 내리고, 촉매가 명확한 종목을 상위로 올림

#### 2-6. 종목 스코어카드 (후보 압축)
2-1~2-5 결과를 바탕으로 각 종목을 아래 항목으로 **간단히 점수화**하여 상위 5~7개 선별:

| 항목 | 배점 |
|------|------|
| 실적 성장 (YoY +10% 이상) | 2점 |
| 재무 건전성 통과 | 2점 |
| 밸류에이션 저평가 | 2점 |
| 기술적 위치 양호 (이평선 위) | 1점 |
| 명확한 촉매 존재 | 2점 |
| 증권사 커버리지 3개사 이상 | 1점 |

- **8점 이상** → 3단계 분석 대상 (우선순위 상)
- **6~7점** → 3단계 분석 대상 (우선순위 하)
- **5점 이하** → 이번 회차 제외, 제외 사유 메모

#### 2-7. 2단계 출력 (3단계 입력값)
2단계 완료 후 반드시 아래 항목을 정리하여 3단계로 넘긴다:
- **후보 종목 5~7개 목록:** 종목명 / 현재가 / 스코어 / 핵심 투자 포인트 한 줄
- **제외 종목 및 사유:** 왜 탈락했는지 한 줄씩 명시
- **주의 종목 (있을 경우):** 권리변경·공시 이슈 등 리스크 플래그

### 3단계: 수급 및 투자 심리 분석 (Sentiment Check)

> **목표:** 2단계 후보 종목(5~7개)의 수급 구조와 투자 심리를 검증하여 **최종 3개 종목**을 확정한다.
> 좋은 펀더멘털도 수급이 무너지면 중기에 못 버틴다 — 수급이 이야기를 완성하는 단계.

#### 3-1. 기관 수급 분석
```bash
python scripts/naver_finance.py investor [종목코드]
# 출력: 날짜 / 종가 / 등락 / 거래량 (최근 20거래일)
# → 외국인·기관 매매 방향을 날짜별로 확인
```
기관의 **매수 지속성과 규모**를 다음 3개 기간으로 교차 확인:

| 기간 | 확인 포인트 |
|------|------------|
| 최근 5일 | 단기 집중 매수 여부 (급등 직전 포지션 진입 신호) |
| 최근 20일 | 중기 누적 순매수 방향 (추세 신뢰도) |
| 최근 60일 | 기관의 장기 관심 여부 (저점 분할 매수 패턴) |

- **긍정 신호:** 3개 기간 모두 기관 순매수 → 강한 수급 뒷받침
- **주의 신호:** 5일만 매수, 60일은 매도 → 단기 반등 매수 가능성, 중기 신뢰도 낮음
- **프로그램 매매 비중** 확인: 프로그램 비차익 매수가 늘고 있다면 기관 ETF 편입 기대 가능

#### 3-2. 외국인 수급 분석 (네이버 증권 [외국인 매매동향])
- **외국인 지분율 추이:** 최근 1개월 지분율 증가 여부 확인
- **순매수 전환 시점:** 외국인이 순매도→순매수로 전환한 시점이 최근 2주 이내인지
- **외국인 한도소진율:** 80% 이상이면 추가 매수 여력 제한 → 리스크 요인으로 기록
- **쌍끌이 매수 판단:** 기관 + 외국인이 **동일 기간 동시 순매수** 중이면 최우선 선정 대상

#### 3-3. 공매도 및 대차잔고 확인
```bash
python scripts/naver_finance.py short [종목코드]
# 또는 브라우저: https://finance.naver.com/item/short_sell.naver?code={종목코드}
```
- **공매도 비율:** 일평균 거래대금 대비 5% 초과 시 하방 압력 존재 → 감점 요인
- **대차잔고 증감:** 최근 2주간 대차잔고 **감소 추세**이면 숏커버링 기대 → 긍정 신호
- **공매도 과열 종목 지정 여부:** 지정 직후는 단기 반등 가능하나 중기 리스크로 기록

#### 3-4. 개인 투자자 심리 진단 (과열 필터)
과열·공포 양 극단을 걸러내는 역할:

**① 과열 경고 신호 (해당 시 선정 보류)**
- 네이버 증권 [종목토론실] 분위기가 "무조건 오른다" 식의 집단 낙관론 지배
- 신용잔고 급증 (최근 1개월 +30% 이상) → 반대매매 위험 구간
- 커뮤니티·뉴스에서 해당 종목이 **이미 핫이슈**로 과도하게 소비된 상태

**② 저평가 심리 신호 (긍정 — 중기 매수 적기)**
- 종목토론실이 조용하거나 약간의 실망 분위기 → 아직 대중이 발견하지 못한 상태
- 신용잔고 안정적 or 소폭 감소 추세
- 최근 뉴스량이 적고 기관만 조용히 매수 중인 패턴

#### 3-5. 수급 종합 스코어카드
3-1~3-4 결과를 합산하여 2단계 스코어에 가산:

| 항목 | 배점 |
|------|------|
| 기관 3개 기간 모두 순매수 | 3점 |
| 외국인 순매수 전환 (최근 2주) | 2점 |
| 쌍끌이 매수 확인 | 2점 |
| 공매도 비율 낮음 + 대차잔고 감소 | 1점 |
| 개인 심리 과열 없음 | 1점 |
| 신용잔고 안정적 | 1점 |

- 2단계 스코어 + 3단계 스코어 **합산 14점 이상** → 최종 선정 0순위
- **10~13점** → 선정 검토 (3개 슬롯 여유 시 포함)
- **9점 이하** → 이번 회차 제외

#### 3-6. 최종 3개 종목 확정 및 출력 (4단계 입력값)
3단계 완료 후 아래 항목을 정리하여 4단계로 넘긴다:
- **최종 선정 3개 종목:** 종목명 / 현재가 / 합산 스코어 / 수급 특이사항
- **탈락 종목 및 사유:** 수급 문제로 탈락한 이유 한 줄 명시
- **수급 리스크 플래그:** 선정 종목 중에서도 공매도·신용 등 주의할 점 기록

### 4단계: 분석 보고서 생성 (Report Generation)

> **목표:** 1~3단계에서 수집한 모든 데이터를 통합하여, 실전에서 바로 활용 가능한 **구조화된 중기 투자 보고서**를 작성한다.

#### 4-1. 보고서 작성 전 최종 정합성 체크 (Data Freeze)
보고서 확정 직전, 아래 항목을 **한 번 더 최신화**하여 오래된 데이터가 보고서에 섞이지 않도록 한다:
```bash
# 선정 3개 종목 현재가 일괄 재확인
python scripts/naver_finance.py stock [종목코드1]
python scripts/naver_finance.py stock [종목코드2]
python scripts/naver_finance.py stock [종목코드3]
```
- 선정 3개 종목의 **현재가 재확인** (★ 현재가 기준 시각 확정 줄 보고서에 기재)
- 최근 48시간 내 **신규 공시·뉴스** 발생 여부 (예상치 못한 악재 필터)
- 증권사 목표주가 **최신 리포트 날짜** 확인 (3개월 초과 리포트는 신뢰도 경고 표시)
- 이상 없을 경우 → 보고서 작성 시작 / 이상 발견 시 → 해당 종목 재검토 또는 후보군에서 대체

#### 4-2. 보고서 헤더 (시장 국면 요약)
보고서 최상단에 **현재 시장 맥락**을 먼저 제시하여 독자가 종목 분석 전에 시장 판단을 공유할 수 있도록 한다:
```
# 제네시스 중기 유망종목 보고서
📅 작성일: YYYY-MM-DD  |  기준가 확인 시각: HH:MM

## 현재 시장 국면
- KOSPI / KOSDAQ 상태: (1단계 국면 판단 결과)
- 주도 섹터: (선정 섹터명)
- 핵심 모멘텀: (3~6개월 지속 근거 한 줄)
- 주요 리스크: (매크로 변수 또는 시장 경고)
```

#### 4-3. 종목별 상세 분석 섹션 (3개 종목 × 동일 포맷)
각 종목을 아래 **7개 블록** 포맷으로 일관되게 작성한다:

**[블록 1] 종목 개요**
- 종목명 / 종목코드 / 업종 / 시가총액
- 현재가 (확정 시각 명시) / 52주 최고·최저가
- 합산 스코어: X점 / 20점 (2단계 + 3단계 합산)

**[블록 2] 투자 Thesis (핵심 스토리)**
- 이 종목을 지금 사야 하는 이유를 **2~3줄**로 압축
- 섹터 모멘텀 → 종목 연결 고리를 명확히 서술
- 예: "AI 서버 수요 급증 → 국내 PCB 업체 중 유일하게 고다층 기판 양산 능력 보유 → 2026년 하반기 대규모 수주 본격화 예상"

**[블록 3] 가격 전략**
```
현재가:      X,XXX원
목표가:      X,XXX원  (+XX%)   ← 증권사 컨센서스 or 목표 PER 기반
1차 손절가:  X,XXX원  (-XX%)   ← 주요 지지선 or 60일선 하단
2차 손절가:  X,XXX원  (-XX%)   ← 구조적 지지선 이탈 시 완전 정리
기대 수익:손실 비율 (R/R): X : 1
권장 보유 기간: 중기 3~6개월
```
- **목표가 산출 근거:** 증권사 컨센서스 평균 / 섹터 평균 PER 적용 / 역사적 PBR 상단 중 사용 기준 명시
- **손절가 설정 원칙:** 1차는 기술적 지지선(-10~15%), 2차는 투자 thesis 훼손 수준(-20%)

**[블록 4] 펀더멘털 요약**
| 항목 | 수치 | 평가 |
|------|------|------|
| 영업이익 YoY | +XX% | ✅ / ⚠️ / ❌ |
| ROE | XX% | ✅ / ⚠️ / ❌ |
| 부채비율 | XX% | ✅ / ⚠️ / ❌ |
| PER (섹터 대비) | XX배 (섹터 XX배) | 저평가 / 적정 / 고평가 |
| 증권사 목표주가 괴리율 | +XX% | — |

**[블록 5] 수급 요약**
- 기관 순매수: (5일 / 20일 / 60일 방향)
- 외국인: (순매수 전환 여부 + 지분율 변화)
- 쌍끌이 여부: ✅ 확인 / ❌ 미확인
- 공매도 비율: XX% (경고 여부)
- 투자 심리: (과열 / 중립 / 저평가 구간)

**[블록 6] 핵심 촉매 이벤트 (Catalyst)**
- 향후 3~6개월 내 주가를 움직일 구체적 이벤트를 **우선순위 순**으로 나열
  - (예: 1) 2026년 Q2 실적 발표 — 어닝 서프라이즈 가능성 高)
  - (예: 2) 정부 XX 정책 예산 배정 확정 예정 (2026년 5월))
  - (예: 3) 신규 수주 계약 공시 기대 (현재 협상 중 보도))

**[블록 7] 리스크 요인**
- 이 종목의 투자 thesis를 훼손할 수 있는 **시나리오**를 2~3개 명시
  - (예: 미-중 관세 전쟁 심화 → 수출 수요 급감)
  - (예: 금리 인상 재개 → 밸류에이션 멀티플 축소)
- 각 리스크에 대한 **대응 기준** 한 줄 추가 (어떤 신호가 나오면 손절할 것인지)

#### 4-4. 포트폴리오 관점 검토
3개 종목 선정 후, 아래 기준으로 **분산 적절성**을 확인한다:
- **섹터 중복 여부:** 3개 종목이 동일 섹터에 몰려있으면 한 종목을 다른 섹터 차상위 후보로 교체 검토
- **시가총액 구성:** 대형주 1개 + 중형주 1~2개 조합 권장 (변동성 균형)
- **모멘텀 타입 혼합:** 실적 성장형 + 수급 주도형 + 정책 수혜형 중 가능하면 2가지 이상 혼합

### 5단계: 결과 검토·배포 및 추적 관리 (Review & Deploy)

> **목표:** 보고서를 사용자에게 전달하고 피드백을 반영한 뒤, GitHub에 배포하고 이후 성과를 추적할 수 있는 상태로 마무리한다.

#### 5-1. 보고서 1차 출력 (사용자 검토용)
- 4단계에서 작성한 보고서를 **대화창에 전체 출력**한다
- 출력 직후 아래 항목을 사용자에게 명시적으로 안내한다:
  ```
  ✅ 보고서 출력 완료

  확인해 주세요:
  1. 현재가·목표가·손절가 수치에 이상이 없는지
  2. 투자 thesis 방향에 동의하시는지
  3. 수정·교체 요청이 있으면 말씀해 주세요

  수정 없으시면 "push 해줘"로 배포 진행합니다.
  ```
- 사용자가 **수정 요청** 시 → 해당 종목·항목만 핀포인트 수정 후 재출력
- 사용자가 **종목 교체 요청** 시 → 3단계 스코어카드의 차순위 후보로 교체

#### 5-2. GitHub 배포 (보고서 생성 완료 즉시 자동 실행)
`push.md` 워크플로우를 즉시 자동 실행한다. 사용자 승인 불필요.

**① 파일 저장 확인**
- `YYYYMMDD-genesis-mid-report.mdx` 파일이 `content/picks/` 경로에 존재하는지 확인
- 없으면 4-5 형식에 맞춰 즉시 생성 후 진행

**② Commit & Push**
```bash
git add content/picks/YYYYMMDD-genesis-mid-report.mdx
git commit -m "feat: [YYYY-MM-DD] 중기 유망종목 보고서 업데이트"
git push origin main
```
- Push 성공 시 → 커밋 해시 및 파일 경로 보고
- Push 실패 시 → 에러 원인 진단 후 해결책 안내

|--------|------------|--------|-----------|------------|------------|
