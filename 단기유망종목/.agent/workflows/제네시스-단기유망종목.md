# 🚀 AntiGravity 단기 유망 종목 발굴 워크플로우 (제네시스)

사용자가 **"제네시스 하자"** 라고 요청하면 즉시 아래 4단계를 순서대로 실행한다.

> **데이터 수집 방법**: `naver-data-fetcher` 스킬의 Python 코드를 Bash로 실행.
> 로컬 PC(한국 IP)에서 직접 네이버 증권에 접근하므로 차단 없음.
> Python 실행 불가 시 스킬 내 Failover 절차를 따른다.

---

## Step 1. 시장 환경 점검 → 공격/중립/방어 판단

### 1-1. KOSPI / KOSDAQ 지수 수집

`naver-data-fetcher` 스킬 **[Function 1]** 실행 후 아래 표를 채운다.

| 지표 | 수치 | 등락 | 거래대금 |
| :--- | :--- | :--- | :--- |
| KOSPI | | | |
| KOSDAQ | | | |
| USD/KRW | | | WebSearch: `달러 원화 환율 오늘` |

### 1-2. 수급 주체 파악

WebSearch 실행: `코스피 코스닥 {오늘날짜} 마감 시황 외국인 기관 순매수`

- 외국인: 순매수 / 순매도 / 혼조
- 기관: 순매수 / 순매도 / 혼조

### 1-3. 시장 판단 결론 (필수 명시 후 Step 2 진행)

| 판단 | 조건 | 액션 |
| :--- | :--- | :--- |
| **공격** | 외인+기관 쌍끌이, 거래대금 20조↑ | 종목 3개 발굴 |
| **중립** | 수급 혼조, 거래대금 보통 | 종목 3개 발굴 (비중 축소 권고) |
| **방어** | 외인+기관 쌍매도, 지수 하락압력 | 종목 2개로 축소, 리스크 가이드 강화 |

> **결론: [ 공격 / 중립 / 방어 ]** ← 반드시 하나 선택 후 기재

## Step 2. 종목 스크리닝 (수급 + 기술적 타점)

### 2-1. 수급 스크리닝 — 후보군 추출

**① 외국인 순매수 상위** → `naver-data-fetcher` 스킬 **[Function 2]** 실행

- KOSPI 상위 10종목 목록 기록
- KOSDAQ 상위 10종목 목록 기록

**② 기관 순매수 상위** → `naver-data-fetcher` 스킬 **[Function 3]** 실행

- KOSPI 상위 10종목 목록 기록
- **외국인 리스트와 겹치는 종목 → 최우선 후보 지정**

**③ 거래량 급증 종목** → `naver-data-fetcher` 스킬 **[Function 4]** 실행

- 전일 대비 거래량 2배 이상 + 상승 마감 종목만 추출

**④ 주도 테마 확인** → WebSearch: `오늘 상승 주도 테마 섹터 종목`

- 당일 상승률 상위 테마 1~2개 식별, 해당 테마 내 후보 추가

> 위 ①②③④를 종합하여 **후보군 5~7종목** 압축

### 2-2. 기술적 타점 검증 — 후보 종목 개별 확인

각 후보 종목별 `naver-data-fetcher` 스킬 **[Function 5]** 실행

```
종목명 (코드) 검증:
  - 현재가: ___원
  - 최근 5일 흐름: 상승 / 하락 / 횡보
  - 이동평균선: 5일·20일·60일선 정배열 여부 → 정배열만 통과
  - RSI 추정: 거래량 패턴으로 과열 여부 판단 (급등 연속 3일↑ = 제외)
  - 거래량 패턴: 음봉 감소 → 양봉 폭발 여부
```

> 수급 + 기술적 조건 동시 충족 종목 **3~5개로 압축**

## Step 3. 리스크 필터링 및 공시 검증

후보 종목별 `naver-data-fetcher` 스킬 **[Function 6]** 실행 후 아래 체크

### 3-1. 즉시 제외 조건 — 하나라도 해당 시 탈락

- [ ] 최근 5거래일 내 **유상증자·전환사채(CB)·주식 담보대출** 공시
- [ ] **투자경고·투자위험 종목** 지정 이력
- [ ] 최대주주·임원의 **대량 지분 매도** 공시
- [ ] 거래소 **조회 공시 요구** 수신

### 3-2. 모멘텀 유효성 확인

- [ ] 상승 이유(테마·수급)가 **뉴스/공시로 뒷받침**되는가
      → 근거 없는 급등 = 세력 개입 가능성 → 제외
- [ ] 섹터 내 **대장주 지위**인가
      → 2~3등 종목보다 대장주 우선

### 3-3. 최종 3종목 확정

**R:R(수익/리스크 비율) 2:1 이상 종목만 선정**
- 예: 목표가 +10% / 손절가 -5% → R:R = 2:1 통과
- 예: 목표가 +8% / 손절가 -7% → R:R = 1.14 → 제외

> **최종 선정: 3종목 (방어 판단 시 2종목)**

## 🚦 운영 원칙

1. **스킬 우선 실행**: 데이터 수집은 반드시 `naver-data-fetcher` 스킬의 Python 코드를 Bash로 실행한다. 임의로 수치를 추정하거나 생략하지 않는다.
2. **데이터 무결성**: 수집한 수치는 소수점까지 그대로 기록한다. 전일 데이터와 혼동하지 않는다.
3. **가격 정보 필수**: 모든 추천 종목에 **현재가·목표가·손절가**를 명시한다. R:R 2:1 미만 종목은 선정하지 않는다.
4. **시장 판단 우선**: Step 1의 공격/중립/방어 결론이 이후 모든 단계의 기준이다. 방어 판단 시 무리한 발굴을 하지 않는다.
5. **자동 푸시**: 리포트 저장 완료 후 `보고서_푸시.md`를 즉시 자동 실행한다.
